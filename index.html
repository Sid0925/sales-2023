fact-dimension Data Model

In my project, I implemented a fact-dimension model to structure and organize the data for analysis. The fact-dimension model is a popular approach used in data modeling, particularly in business intelligence and data warehousing. In this model, I designed a fact table that contains the primary business metrics or facts that are the focus of analysis. These facts could include measures like revenue, quantity sold, profit, or any other key performance indicators (KPIs) relevant to the business. Alongside the fact table, I created dimension tables that provide additional context and descriptive attributes related to the facts. Dimension tables represent various aspects of the business, such as customers, products, time, locations, or any other relevant dimensions that help analyze and filter the facts. The dimension tables are linked to the fact table through relationships, typically using primary and foreign keys. This allows for easy and efficient querying and slicing and dicing of the data based on different dimensions and their hierarchies. By separating the facts and dimensions, the fact-dimension model promotes data integrity, scalability, and flexibility in analysis. By implementing the fact-dimension model, I was able to create a structured and organized data model that facilitated efficient data analysis and reporting. It allowed for granular analysis at different levels of detail, such as by customer, product, or time, providing valuable insights into the business's performance and enabling data-driven decision-making.

DAX And
Calculated Columns

Calculated columns are an essential feature in data modeling, allowing for the creation of new derived columns based on existing data using formulas or expressions. These columns are computed during the data loading or transformation process and become part of the data model. By leveraging calculated columns, I was able to introduce additional metrics or calculations that were not directly available in the raw data. For example, I utilized calculated columns to determine the total revenue by multiplying the quantity sold with the price per unit.

DAX (Data Analysis Expressions) is a powerful formula language used in various analytical tools, including Power BI. It enables the definition of custom calculations, measures, and aggregations within the data model. With DAX, I could perform complex calculations and advanced data manipulations. By utilizing DAX functions such as SUM, AVERAGE, MAX, MIN, and IF, I could aggregate data, apply business logic, and perform conditional calculations. For instance, I utilized the AVERAGE function in DAX to calculate the average order quantity.

Intricate DAX Expressions: A Step-by-Step Guide

In this guide, we'll explore some complex DAX expressions used to derive valuable insights and KPIs. I'll walk you through the logic and functions used in each measure and calculated column, explaining them in a step-by-step manner.

Measures

1. Running Total

The Running Total measure calculates the cumulative revenue for the last 10 days from the maximum date in the data. Here's how we do it:

  1. Find the Maximum Date: Using the MAX function, we identify the most recent date in the 'Calendar_dimension' table.
  2. Calculate the Date Range: The DATESINPERIOD function defines the last 10 days from the maximum date.
  3. Compute Running Total: The CALCULATE function sums up the "Total Revenue" within the specified period, resulting in the Running Total measure.

2. TOP5 Products

The TOP5 measure identifies the top 5 products based on their revenue. Here's the step-by-step process:

  1. Select All Products: The ALLSELECTED function includes all selected products in the filter context.
  2. Sort by Revenue: The TOPN function retrieves the top 5 products based on "Total Revenue" in descending order.
  3. Calculate Total Revenue: The CALCULATE function computes the "Total Revenue" for the selected top 5 products, preserving existing filters with KEEPFILTERS.

3. High Ticket Orders

The High Ticket Orders measure calculates the number of orders with a price greater than the overall average price. Here's how:

  1. Filter Products: The FILTER function filters products where the "ProductPrice" is greater than the "Overall Average price."
  2. Calculate Total Orders: The CALCULATE function sums up the "Total Orders" for the filtered products.

4. Cumulative Total Revenue

The Cumulative Total Revenue measure calculates the revenue up to a selected date. Here's the detailed explanation:

  1. Select All Dates: The ALLSELECTED function includes all selected dates in the filter context.
  2. Filter Dates: The FILTER function includes dates less than or equal to the maximum selected date.
  3. Calculate Cumulative Revenue: The CALCULATE function sums up the "Total Revenue" for the selected dates.

Calculated Column Transformation

1. Customer Priority

The "Customer Priority" column classifies customers as "Priority" or "Standard" based on their annual income and whether they are parents. Here's how:

  1. Set Conditions: The AND function checks if the annual income is greater than $100,000 and if the customer is a parent.
  2. Assign Priority: The IF function sets the priority as "Priority" if the conditions are met, otherwise "Standard."

2. Income Level

The "Income Level" column classifies customers into different income categories. Here's the detailed process:

  1. Set Income Ranges: The SWITCH function checks the annual income and classifies it into "Very High," "High," "Average," or "Low."
